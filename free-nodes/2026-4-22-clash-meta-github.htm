第二章 手把手安装教程

2.1 应用获取与权限管理

在Google Play或App Store搜索"Surfboard"时，需注意辨别正版应用（开发者通常标注为"Surfboard LLC"）。安装完成后，系统会要求授予三项关键权限：
- 网络访问权限（必须允许）
- 本地存储权限（用于配置文件缓存）
- 自启动权限（保持后台连接稳定）

专业建议：在MIUI等深度定制系统上，需额外在"电池优化"设置中将Surfboard设为"不限制"，防止系统自动清理后台进程。

2.2 首次运行配置向导

启动应用后，新手引导界面会提供三种预设模式：
- 极速模式（优先速度，适合视频/下载）
- 平衡模式（速度与安全兼顾）
- 隐私模式（最高安全等级，速度略有牺牲）

建议初次使用者选择"智能推荐"，系统会根据当前网络环境自动优化参数。

第三章 深度配置进阶指南

3.1 服务器选择艺术

点击"服务器列表"会显示200+个节点，关键筛选指标包括：
- 延迟值（＜100ms为优质节点）
- 负载状态（避免选择＞70%负载的节点）
- 地理位置（物理距离每增加1000km，延迟增加约20ms）

实测技巧：长按节点可进行ping测试，连续三次测试取平均值更准确。

FAQ

Mellow 是否支持 IPv6 流量代理？
Mellow 支持 IPv6，只需在配置文件中启用 IPv6 DNS 解析并确认操作系统开启 IPv6 路由功能。部分节点若不支持 IPv6，则相关请求会自动回退到 IPv4，保证兼容性。
Trojan 伪装域名如何选择以提升隐蔽性？
伪装域名应选择真实可访问网站并配备有效证书，这样 Trojan 流量可模拟正常 HTTPS 请求，降低被封锁和检测的风险，提高节点长期可用性和访问稳定性。
SSR 的混淆插件如何增强隐蔽性？
混淆插件可模拟正常协议特征，如 HTTP 或 TLS 流量，使 SSR 流量难以被检测。合理选择混淆方式和协议，可提升在高封锁环境下的长期稳定性和安全性。
V2RayA 在 Linux 系统中如何开机自启？
可通过 systemd 创建启动项或在启动脚本中添加 V2RayA 启动命令。编辑 /etc/systemd/system/v2raya.service 文件并设置 `Restart=always`，执行 `systemctl enable v2raya` 使其随系统自动启动。
QuantumultX 的广告屏蔽原理是什么？
QuantumultX 使用分流规则与脚本结合的方式实现广告屏蔽。通过重写 HTTP 请求和阻止特定域名，它能有效去除 App 内广告或网页弹窗，提升浏览体验。
Sing-box 的自定义 outbound 功能有什么用？
Sing-box 允许用户自定义多个 outbound 出口，并通过路由规则选择性输出。例如可让流媒体走特定节点，而其他网站使用备用节点，从而实现精准分流。
如何提高 Vmess 协议的安全性？
可通过启用 TLS 加密、使用安全端口（如 443）、结合 WebSocket 或 gRPC 传输方式来提升安全性。同时，避免公开分享 UUID，定期更换节点参数能进一步减少被封风险。
VMess AEAD 加密为何适合高封锁环境？
AEAD 加密可防止数据篡改和流量特征泄露，降低被检测风险。在高封锁网络中，保证节点可用性和连接稳定性，同时提升数据传输安全性，确保长期安全访问。
QuantumultX 支持按端口和协议分流吗？
支持。QuantumultX 可以按 TCP、UDP、端口和应用协议分流流量，结合策略组使用，确保游戏、视频和广告屏蔽走最优节点，提高访问速度和网络稳定性。
Hiddify Next 如何为不同用户设置流量限制？
管理员可为每个用户设置独立带宽限制和流量配额。超过限制时，可自动降速或阻止访问，确保服务器资源合理分配，避免单用户占用过多带宽，提升整体网络效率和公平性。
